# Formal Languages & Word Operations in C  

---

## Overview

This project implements the foundational concepts of **formal language theory** in pure C — no external libraries, no `<string.h>`. It covers everything from basic word operations over character-based strings to a fully alphabet-aware word model with an automaton validator, all unified into a single interactive console application.

The core idea: a **word** is not just a string of characters — it is a sequence of **symbols** drawn from a defined **alphabet**. This distinction is what separates formal language theory from ordinary string processing, and it is exactly what this program models.

---

## Features

### Alphabet System
- Define a custom alphabet with any number of symbols (including **multi-character symbols** like `ab`, `aa`, `0`, `1`)
- Duplicate detection on input
- Greedy longest-match parser to tokenize raw strings into symbol sequences

### Word Operations (all alphabet-aware)

| # | Operation | Description |
|---|-----------|-------------|
| 1 | **Length** | Counts the number of symbols in the word (recursive) |
| 2 | **Count Occurrences** | Counts how many times a given symbol appears (recursive) |
| 3 | **Concatenation** | Joins two words: `w1.w2` |
| 4 | **Power** | Repeats a word n times: `wⁿ` |
| 5 | **Equality** | Checks if two words are identical (recursive) |
| 6 | **Mirror** | Reverses the symbol order of a word |
| 7 | **Palindrome** | Checks if a word reads the same forwards and backwards (recursive) |

### Automaton Validator
- Enter any raw string and the program tells you whether it is **ACCEPTED** or **REJECTED** by the current alphabet
- Acts as a simple recognizer (finite automaton concept): can the string be fully decomposed into valid symbols?
- Pinpoints the exact position of failure if the string is rejected

---

## How It Works

### Word Representation
Words are stored as arrays of **symbol indices** (not characters), referencing positions in the alphabet table:

```
Alphabet = { "a", "ab", "b" }
Raw input: "abab"
Parsed word: [ "ab", "ab" ]   ← greedy longest-match, length = 2
```

### Parser (Greedy Longest-Match)
The parser scans the raw string left-to-right and at each position picks the **longest symbol** in the alphabet that matches. This ensures unambiguous tokenization.

## Concepts Covered

- **Alphabet (Σ)** — finite set of symbols
- **Word / String** — finite sequence of symbols from Σ
- **Empty word (ε)** — word of length zero
- **Language** — set of words over an alphabet
- **Concatenation** — `uv` is `u` followed by `v`
- **Power** — `wⁿ` = w concatenated with itself n times; `w⁰ = ε`
- **Mirror / Reverse** — `w̃` reverses symbol order
- **Palindrome** — `w = w̃`
- **Automaton (recognizer)** — determines membership of a string in a language
